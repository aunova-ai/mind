## 1. 아키텍처 요약

- **관리 방식**: 구글 시트에서 전체 문항 통합 관리
- **배포 방식**: 월별 JSON 파일로 분할 (`content/questions/2026_MM.json`)
- **문항 규모**: 월 110개 (객관식 100 + 주관식 10)
- **연간 총량**: 1,320문항 (12개월)
- **웹앱 로딩**: 현재 월 JSON만 호출 → 약 50~60KB, 로딩 0.1초 미만

## 4. 10대 분류 카테고리 (5 Domain x 2 Metric 하이브리드)

| ID | 분류 영역 (Domain) | 기상 특성 (Metric) | 설명 | 속성 |
| :--- | :--- | :--- | :--- | :--- |
| **CAT-01** | 개인 (Personal) | ☀️ **날씨** (활력) | 개인의 활력 (아침 기상, 자기계발 에너지) | 능동 에너지 |
| **CAT-02** | 개인 (Personal) | 💧 **습도** (자책) | 개인의 자책 (부정적 자아상, 자존감 저하) | 수동적 우울 |
| **CAT-03** | 대인 (Interpersonal) | ☀️ **날씨** (사회성) | 관계의 활력 (모임에서의 에너지, 외향성) | 능동 에너지 |
| **CAT-04** | 대인 (Interpersonal) | 💧 **습도** (피로도) | 관계의 피로도 (대화 후의 소모감, 타인 의식) | 수동적 불안 |
| **CAT-05** | 심리 (Psychological) | ☀️ **날씨** (여유) | 정서적 여유평상시의 마음 평화, 잔잔함) | 능동 에너지 |
| **CAT-06** | 심리 (Psychological) | 💧 **습도** (불안) | 정서적 불안감 (막연한 무기력, 감정 널뛰기) | 수동적 우울 |
| **CAT-07** | 상황 (Situational) | ☀️ **날씨** (회복력) | 위기 극복, 회복 탄력성 (긍정적 대처) | 능동 에너지 |
| **CAT-08** | 상황 (Situational) | 💧 **습도** (취약성) | 스트레스 취약성 (돌발 상황 시 무력감, 패닉) | 수동적 불안 |
| **CAT-09** | 미래 (Future) | ☀️ **날씨** (낙관) | 미래 낙관성 (내일에 대한 설렘과 기대감) | 능동 에너지 |
| **CAT-10** | 미래 (Future) | 💧 **습도** (막막함) | 미래 비관성 (앞날에 대한 폐쇄적 절망감) | 수동적 우울 |
| **CAT-SUB** | 심화 (Deep-Dive) | 📝 **주관식** | 감정 일기 (가장 차오른 습도의 원인 해소) | 치유 / 기록 |

## 5. 월별 110문항 컨텐츠 운영 규칙

- **객관식 100문항**: 10개 분류 × 각 10문항 (분류 내 문항 스타일 중복 금지)
- **주관식 10문항**: 분류 구분 없이 깊은 내면 질문으로만 구성
- **랜덤 출제 규칙**: 앱 실행 시 10개 분류에서 각 1문항씩 랜덤 선택 → 총 10문항 출제
- **중복 방지**: 동일 사용자에게 같은 문항이 연속 출제되지 않도록 로컬 세션 기록 활용
