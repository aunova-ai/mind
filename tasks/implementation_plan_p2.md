# 📑 단계 2 실행 계획: 콘텐츠 자동화 프로토타입 (마인드웨더) - 제로-코스트 버전

본 문서는 Amanager(Antigravity)에 의해 설계된 **[제로-코스트(Zero-Cost)]** 중심의 실행 지침입니다. 유료 API(DALL-E, GPT API 등) 사용을 배제하고, 미리 생성된 에셋과 로컬 로직을 연동하여 비용 없는 대량 배포 기반을 마련합니다.

---

## 💎 P2-1. 심리 트리거 및 정적 데이터셋 구축
> **목표**: AI 실시간 생성 대신, 검증된 텍스트 뱅크를 구축하여 비용을 제거한다.

1.  **결과 유형별 고정 답변 (Fixed Results)**: 4가지 결과(맑음, 흐림, 폭풍, 안개)에 대한 정교한 진단 텍스트 및 솔루션 문구를 미리 작성(Copy 에이전트 담당).
2.  **문항-가중치 매트릭스**: 각 질문의 선택지가 결과 점수에 미치는 영향력을 엑셀/JSON으로 고정 설계.
3.  **데이터 구조화**: `results_content.json` 파일을 생성하여 모든 텍스트 자산을 중앙 집중화.
4.  **소스 검증**: 무료 심리 분석 도구 및 논문 소스를 활용하여 로직의 신뢰성 확보.
5.  **버전 관리**: 콘텐츠 수정 시 파일 교체만으로 반영되도록 설계.

## 💎 P2-2. 정적 로직 기반 웹 인터페이스 (Static Web Engine)
> **목표**: 서버 연산 비용 없는 Vercel 기반의 정적 페이지(Static Export)를 구축한다.

1.  **클라이언트 사이드 로직**: n8n/Make 호출 대신 브라우저(JavaScript)에서 즉시 점수를 계산하고 결과를 출력하는 엔진 구현.
2.  **Vercel 배포**: 무료 티어 범위 내에서 `mind.aunova.ai` 도메인 운영.
3.  **데이터 저장**: 응답 데이터를 외부 유료 DB 대신 Google Sheet(App Script 활용)로 무료 전송/저장.
4.  **경량화 아키텍처**: Next.js의 정적 생성(SSG) 기능을 활용하여 로딩 속도 극대화 및 비용 0원 유지.
5.  **웹훅 프리(Webhook-free)**: 초기 단계에서는 복잡한 서버리스 함수 대신 클라이언트 요청 위주로 구성하여 API 한도 이슈 회피.

## 💎 P2-3. 마케팅 로직 및 전환 포인트 설계
> **목표**: 비용 없는 채널(오픈톡, 블로그)을 통한 유입 및 전환을 설계한다.

1.  **바이럴 헤드라인**: 사용자가 결과 이미지를 갈무리하여 공유하고 싶게 만드는 '인스타 감성' 디자인 적용.
2.  **무료 전환 경로**: 유료 상담 대신 '무료 오픈채팅방' 또는 '정보형 뉴스레터' 구독으로 CTA 연결.
3.  **SNS 최적화 메타 태그**: 공유 시 제목과 이미지가 매력적으로 노출되도록 Open Graph 정밀 설정.
4.  **커뮤니티 바이럴**: 소모임, 문토 등 무료 모집 채널에 적합한 홍보 스크립트 작성.
5.  **데이터 분석**: Google Analytics(무료)를 연동하여 유입 및 이탈 지점 정밀 추적.

## 💎 P2-4. 이미지 에셋 프리-프로덕션 (Asset Pre-matching)
> **목표**: DALL-E 3 실시간 API 비용을 0원으로 만들기 위해 에셋을 선 제작한다.

1.  **배치 생성(Batch Generation)**: Spark 에이전트가 단발성으로 고품질 이미지를 40~100종 미리 생성(DALL-E 무상 쿼터 또는 로컬 스테이블 디퓨전 사용).
2.  **에셋 매트릭스**: 결과 점수(0~100) 및 특정 키워드 조합에 따라 최적의 이미지를 매칭하는 `image_map.json` 구축.
3.  **이미지 호스팅**: GitHub LFS 또는 무료 이미지 호스팅(Cloudinary 무료티어 등)을 통한 저장 및 호출.
4.  **캐릭터 일관성**: 미리 제작된 이미지 군단이 동일한 아트 스타일(Aunova Art-Style)을 유지하도록 사전 검수.
5.  **동적 합성 대체**: 실시간 합성 대신 텍스트가 이미 포함된 여러 버전의 이미지를 준비하여 로딩 부담 경감.

## 💎 P2-5. 시스템 안정화 및 확장 준비
> **목표**: 제로-코스트 모델의 안정성을 검증하고 향후 유료 모델 전환 기반을 마련한다.

1.  **부하 테스트**: 동시 접속자가 몰려도 정적 페이지 특성상 무너지지 않는 안정성 확인.
2.  **데이터 무결성 검사**: Google Sheet로 데이터가 중복 없이 정확히 쌓이는지 확인.
3.  **비용 모니터링**: 0원 유지를 위해 각 서비스(Vercel, Google)의 무료 한도 임계치 도달 여부 상시 체크.
4.  **피드백 수집**: 사용자 만족도 조사를 통해 3단계(미디어 파이프라인)의 우선순위 결정.
5.  **유료 브릿지 설계**: 향후 정밀 진단 등 유료 서비스 도입 시 전환될 수 있는 결제 버튼 미리 배치(비활성 상태).
